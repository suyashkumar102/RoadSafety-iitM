# Submission Requirements

Based on the hackathon information in `info.md`, Stage 1 needs:

- Complete solution code.
- 7-slide presentation including Welcome and Thank You slides.
- Complete code details.
- Software packages used.
- Assumptions document.
- One submission through Unstop for one selected topic.

## RoadSoS package checklist

- `README.md` with setup instructions.
- `plan.md` with execution and merge plan.
- `docs/data_sources.md` with source-backed contact references.
- `docs/assumptions.md` with known limits.
- `demo/golden_scenario.md` with the live demo script.
- Final screenshots for the deck.
- Final test/manual verification notes.

## Assumptions to document before submission

- RoadSoS does not dispatch emergency services directly.
- RoadSoS does not guarantee real-time availability.
- RoadSoS does not provide medical diagnosis or legal advice.
- Contact quality depends on source freshness.
- Offline cache is scoped to curated demo regions.
